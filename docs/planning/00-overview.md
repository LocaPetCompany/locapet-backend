# 00. 서비스 개요 (Service Overview)

> 본 문서는 로카펫(Locapet) 서비스의 **최상위 기획 레이어**로, 이후 도메인 설계 / UI·UX / API 설계의 공통 기준점이 된다.
> 대상 독자: PM, 기획자, 백엔드/클라이언트 개발자, QA, 디자이너.

---

## 1. 서비스 한 줄 소개

> **로카펫(Locapet)은 반려동물과 함께 갈 수 있는 장소(식당·카페·유치원 등)를 찾고, 직접 다녀온 사람들의 리뷰를 공유하는 모바일 서비스다.**

- 내 반려동물의 특성(종·크기·기질)에 맞는 장소를 추천받고,
- 같은 조건의 반려동물 보호자가 남긴 "맞춤리뷰"를 통해 신뢰도 높은 정보를 얻으며,
- 찜·최근 본 장소·저장된 검색 필터로 나만의 반려생활 데이터베이스를 구축한다.

---

## 2. 타겟 사용자

### 2.1 Primary Persona
- **20~40대 반려동물 보호자 (주로 개·고양이)**
- 외출·여행·나들이 시 "반려동물과 함께 갈 수 있는 곳"을 매번 포털 검색으로 찾는 불편을 겪음
- SNS·블로그 후기에 의존하지만, 내 반려동물과 유사한 조건(크기·종)이 아니면 신뢰하기 어려움

### 2.2 Secondary Persona
- **반려견 유치원/호텔/훈련소를 정기적으로 이용하는 보호자** — "로카유치원" 카테고리 타겟
- **반려동물 이벤트·여행을 계획하는 보호자** — 저장된 검색 필터/찜 기능 타겟

### 2.3 관리자(Admin) Persona
- 업체 정보 등록·검수, 리뷰/문의/신고 처리, 공지·팝업·유지보수 관리

---

## 3. 핵심 가치 제안 (Value Proposition)

| # | 가치 | 설명 |
|---|------|------|
| V1 | **검증된 "동반 가능" 정보** | 업체별로 반려동물 동반 가능 여부·조건(크기 제한, 실내 동반, 케이지 필수 등)을 명확히 표시 |
| V2 | **내 반려동물에 맞춘 추천·리뷰** | 등록한 반려동물 프로필과 유사한 조건의 리뷰만 필터링 ("맞춤리뷰") |
| V3 | **장소 탐색의 다축 제공** | 지도·목록·검색·내 주변·카테고리별 진입 지원 |
| V4 | **개인화 저장 공간** | 찜·최근 본 장소·저장된 검색 필터로 반복 탐색 비용 절감 |
| V5 | **로카 큐레이션** | "로카 추천 장소", "요즘 찜 많은 장소" 등 에디토리얼/알고리즘 기반 큐레이션 |

## 4. 주요 Use Case

| UC# | 시나리오 | 관련 도메인 |
|-----|----------|-------------|
| UC1 | **최초 가입 → 반려동물 등록 → 첫 탐색** : 소셜 로그인 → 본인인증(PASS) → 프로필 완성 → 반려동물 등록(10단계) → 홈 진입 | 인증/온보딩, 반려동물 관리, 홈/피드 |
| UC2 | **주말 나들이 장소 찾기** : 홈에서 "내 주변" → 카테고리(식당) 선택 → 지도/목록 전환 → 업체 상세 → 리뷰 확인 → 찜 | 홈/피드, 지도/위치, 업체/상세, 리뷰, 찜 |
| UC3 | **맞춤리뷰로 결정 내리기** : 업체 상세 → 전체 리뷰 보기 → "맞춤리뷰만" 토글(내 반려동물과 유사한 조건) → 방문 결정 | 업체/상세, 리뷰, 반려동물 관리 |
| UC4 | **리뷰 작성** : 방문 후 업체 상세 → 리뷰 작성 → 반려동물 선택(어떤 아이와 방문했는지) → 사진·평점·본문 → 제출 | 리뷰, 반려동물 관리 |
| UC5 | **반복 탐색 효율화** : 검색 → 상세 필터 설정 → "필터 저장" → 다음 방문 시 저장된 필터로 재검색 | 검색/필터 |
| UC6 | **로카유치원 탐색** : 홈 → 로카유치원 진입 → 지도에서 내 주변 유치원 확인 → 상세 → 문의/예약 | 지도/위치, 업체/상세, 고객지원/문의 |
| UC7 | **문제 발생 대응** : 부적절 리뷰 신고 / 업체 오류 제보 / 1:1 문의 작성 → 관리자 답변 확인 | 리뷰, 고객지원/문의, 팝업/모달 |

## 5. 범위 정의 (Scope)

> 기준: "MVP는 핵심 가치 V1·V2·V3을 최소 플로우로 완성할 수 있는 범위" — 발견(Discover) → 확인(Verify) → 기록(Record) 3단계 루프를 닫는 것이 목표.

### 5.1 MVP 포함 (Must Have)

| 도메인 | MVP 범위 | 근거 |
|--------|----------|------|
| 인증/온보딩 | 소셜 로그인(4종) · 본인인증 · 프로필 완성 · 탈퇴 | **이미 구현 완료** (CLAUDE.md 참조) |
| 메타 | 앱 버전 · 유지보수 · 공지 | **이미 구현 완료** |
| 반려동물 관리 | 등록·수정·삭제·대표 설정 | 맞춤리뷰의 필수 전제 |
| 업체/상세 | 상세 페이지 · 기본 정보 · 동반 조건 표시 | 전 도메인의 중심축 |
| 홈/피드 | 기본 홈(추천 섹션) | 첫 진입 화면 |
| 검색/필터 | 기본 검색 · 주요 필터 · **필터 저장은 제외** 가능성 검토 | V3 충족. 저장 필터는 복잡도가 높음 ❓|
| 지도/위치 | 지도 보기 · 목록 전환 · 내 주변 | V3 충족 |
| 리뷰 | 작성·조회·수정·삭제 · **맞춤리뷰 필터** | V2 충족의 핵심 |
| 찜 | 찜 등록·해제·목록 | 재방문 유도 |
| 마이페이지 | 프로필 조회·수정 | 기본 UX |
| 알림/공지 | 공지사항 · FAQ (**푸시알림은 Phase 2 검토**) | 운영 최소 |
| 고객지원/문의 | 1:1 문의 작성·조회 | 운영 최소 |
| 팝업/모달 | 공통 Dialog/Modal · 신고하기 | 안전장치 |

### 5.2 Phase 2 이후 (Nice to Have / Later)

| 항목 | 이유 |
|------|------|
| 최근 본 장소 | 필요하나 찜으로 대체 가능, 운영 데이터 쌓인 후 고도화 |
| 저장된 검색 필터 (22개 화면의 복잡도) | UX 복잡도 크고 사용률 데이터 필요 |
| 예약 기능 | 외부 연동/파트너십 필요 ❓ |
| 푸시 알림 | FCM 연동 + 정책 수립 필요 |
| 업체 전화/이메일/카톡채널 연동 팝업 | 업체 데이터 적재 정책과 연동 |
| 로카 추천 큐레이션 엔진 | 운영자 수동 큐레이션 → 추천 알고리즘 단계적 도입 |
| 리뷰 신고 자동화(AI 필터) | 수동 운영 → 축적 데이터 기반 자동화 |

### 5.3 범위 **밖**(Out of Scope)

- 반려동물 보험/쇼핑몰/커머스 기능
- 보호자 간 직접 메시징/커뮤니티(게시판)
- 반려동물 헬스케어/건강기록

## 6. 비기능 요구사항 개요

### 6.1 성능
- 홈 진입 시 TTI 3초 이내 (메타 API + 홈 피드 병렬 호출)
- 지도 목록 로딩: 100건 기준 1초 이내 응답
- 검색/필터 API: 필터 적용 후 500ms 이내 응답 (p95)
- Redis 캐시로 스플래시/메타/공지 TTL 5분 기본 적용

### 6.2 보안
- 모든 API는 HTTPS 필수
- JWT 기반 인증 (accessToken 30분, refreshToken 14일) — 이미 구현
- PASS 본인인증 CI 해시는 HMAC-SHA256으로 저장 (원본 CI 저장 금지) — 이미 구현
- 업로드 이미지는 signed URL 기반 업로드 (직접 S3 노출 금지) ❓구체 정책 결정 필요
- 민감정보(이름·생년월일·전화번호) 마스킹 기본 정책 수립 필요

### 6.3 확장성
- 모듈러 모놀리스 (app-api + admin-api + domain + common) 구조 유지
- 업체·리뷰 도메인은 추후 읽기 전용 Replica 분리 가능하도록 설계
- 지도 좌표 검색은 PostGIS 또는 Redis Geo 도입 검토 ❓
- 이미지는 S3 + CloudFront 조합 (또는 외부 CDN) ❓

### 6.4 데이터 정합성
- 모든 시각은 UTC로 저장·전송 (`Instant` / `TIMESTAMPTZ`) — 이미 구현
- 탈퇴 30일 유예 정책 — 이미 구현
- 맞춤리뷰 필터링의 기준(반려동물 속성) 변경 시 이력 추적 필요 ❓

### 6.5 운영성
- `admin-api` 분리로 관리자 기능 격리
- Flyway 기반 마이그레이션, JPA `validate`만 사용
- 공지/유지보수/앱버전 게시는 관리자 DB 조작만으로 즉시 반영 가능 (캐시 무효화 정책 필요) ❓
